# snap_qc
code and data for modeling SNAP payment errors

Putting this out on my github with an Apache 2.0 license as an assurance that anyone can freely use and build upon the code, the approaches it uses, and the results. The code itself is quite a mess - I do not recommend trying to use it as-is. The goal of this repo is to make it unambiguous that anyone can freely use ideas / materials they've seen me present regarding SNAP QC. I'll revise the code to be easier to read and work out of the box if there's demand for that. 

Please note that these scripts rely heavily on Claude Code for nearly all of the key functions. I've left the code I wrote to manually test these functions in the scripts, which hurts readability, but I wanted to provide some assurance that there's been a lot of QA that went into whether the functions worked as intended.
