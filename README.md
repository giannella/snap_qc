# snap_qc
code and data for modeling SNAP payment errors

Putting this out on my github as an assurance that anyone can freely use and build upon the code, the approaches it uses, and the results. The code itself is quite a mess and I'll revise and remove this message if there's sufficient demand for something that works well out of the box.

I've left the code I wrote to manually test these functions in the scripts, which hurts readability, but I wanted to provide some assurance that there's been a lot of QA that went into whether the functions worked as intended.
